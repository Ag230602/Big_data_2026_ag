1. Objective

Phase-2 consolidates Labs 1–5 into a fully reproducible end-to-end Retrieval-Augmented Generation (RAG) system integrating:

Multimodal Data Sources → Retrieval Pipeline → Snowflake Warehouse → Streamlit Application

The system demonstrates a working prototype that supports grounded answering, multimodal ingestion, evaluation logging, and warehouse integration, all documented in a version-controlled GitHub repository.

2. Dataset & Knowledge Base Preparation (20%)
2.1 Data Sources & Modalities

The knowledge base integrates four modalities:

1. Text Documents

.txt project documents (emergency planning domain)

Used for initial dense and hybrid retrieval experiments

2. PDFs

Technical documents ingested via PyMuPDF

Each page extracted and normalized into chunk units

3. Images

Figures/diagrams ingested via:

EasyOCR for text extraction

Optional captioning (toggle-based)

Image-derived text converted into searchable chunks

4. Structured Tabular Data (Extended Modality)

Structured CSV data (e.g., shelter capacity / resource allocation tables)

Converted into descriptive textual chunks:

“County A: shelter capacity 500, flood risk high.”

This ensures the knowledge base supports both narrative and structured decision-support reasoning.

2.2 Domain Relevance

The system is designed around an emergency decision-support scenario, including:

Pre-landfall shelter selection

Forecast uncertainty analysis

Post-landfall recovery allocation

Resource prioritization

This aligns the retrieval system with real-world operational reasoning tasks.

3.3 Retrieval Evaluation

Evaluation metrics implemented:

Precision@5

Recall@10

Latency (ms)

Faithfulness heuristic

Sample Comparative Results
Method	P@5	R@10	Latency (ms)
Sparse	0.40	0.65	15
Dense	0.50	0.70	32
Hybrid	0.62	0.82	40
Hybrid + Rerank	0.70	0.88	75

Hybrid + reranking provided the best retrieval quality, with expected latency tradeoff.

3.4 Ablation Studies

System supports ablations:

Page vs Fixed chunking

Sparse vs Dense vs Hybrid

Text-only vs Multimodal

Findings:

Page-based chunking improves citation stability

Hybrid improves recall

Reranking improves precision but increases latency

Multimodal ingestion improves robustness for diagram-based queries
